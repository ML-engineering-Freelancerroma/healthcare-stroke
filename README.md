# Исследование влияния SMOTENC и RandomUnderSampler на задачу бинарной классификации при сильном дисбалансе классов

Этот репозиторий содержит небольшое исследование влияния методов **SMOTENC** и **RandomUnderSampler** (библиотека `imblearn`) на данные с выраженным дисбалансом целевого класса в задаче бинарной классификации.

Датасет: [Healthcare Dataset Stroke Data](https://www.kaggle.com/datasets/aouatifcherdid/healthcare-dataset-stroke-data)

## Предобработка и обработка данных

- Данные разделены на валидационную и тестовую выборки в соотношении **75:25** с сохранением пропорции целевого класса  
→ файл: `healthcare-stroke-prepare-for-ML.ipynb`

- Выполнена обработка признаков различными способами:  
  - one-hot encoding  
  - label encoding  
  - ручное кодирование  
→ файл: `feat_prepare.ipynb`

## Важные метрики

Для задач такого типа приоритетными являются:

- **Recall**  
- **ROC-AUC**  
- **PR-AUC**  

(важно выявить как можно больше истинно положительных случаев — пациентов с риском инсульта с контролем ложных срабатываний)

→ файл: `train_SMOTENC.ipynb`

## Выводы по результатам метрик тренировочных данных

По большинству метрик **RandomUnderSampler** показал себя лучше при стандартных/типовых настройках моделей.

Более тонкая настройка гиперпараметров может дать заметно лучший результат, но требует значительных вычислительных ресурсов и времени.

Проведено тестирование на отложенной тестовой выборке (данные, не участвовавшие в обучении и подборе гиперпараметров). Проведено сравнение метрик получившихся данных и тестовых (submission_test_data vs healthcare-sub).

→ файл: `test_data.ipynb`

## Выводы по результатам метрик тестовых данных

Recall (0.8065) - модель смогла обнаружить более 80% всех реальных случаев инсульта на тестовых данных, риск пропустить опасный случай относительно невысок.
Accuracy (0.6659) - достаточно низкое значение, что ожидаемо при высоком Recall в условиях сильного дисбаланса классов. Модель часто предсказывает положительный класс, что приводит к большому количеству ложных срабатываний и снижению общей точности.
ROC-AUC (0.8365) - модель хорошо ранжирует бинарный класс.
PR-AUC (0.2429) - средний показатель, при высоком Recall, Precision заметно падает, что естественно сказывается на PR-AUC. Это типично для агрессивных моделей с низким порогом классификации.
Поскольку наиболее важной метрикой был выбран Recall, можно утверждать, что модели отлично справились с задачей бинарной классификации при сильном дисбалансе классов.

## Установка

1. Склонируйте репозиторий:
```
git clone https://github.com/ML-engineering-Freelancerroma/healthcare-stroke.git
```
2. Перейдите в локальную директорию, создайте виртуальное окружение и активируйте его:
```
cd healthcare-stroke
python -m venv venv
source venv/Scripts/activate
```
3. Установите зависимости:
```
pip install -r requirements.txt
```
